# .ai/ — База знаний AI-агентов проекта Tariff

Эта директория является точкой входа для любого AI-агента, подключающегося к проекту.  
Начни именно здесь. Прочитай документы в порядке, указанном ниже.

---

## Порядок чтения

### 1. `ENGINEERING_PROTOCOL.md` — **читать первым, всегда**
Обязательный регламент работы. Содержит правила, которые нельзя нарушать:
- принцип Root Cause Analysis перед любым изменением
- принцип минимальных изменений
- правила изменения API, БД, Frontend
- Definition of Done
- формат итогового отчёта после каждой задачи

> Если ты не прочитал этот файл — ты не готов к работе в проекте.

---

### 2. `PROJECT_CONVENTIONS.md` — архитектура и соглашения
Описывает устройство проекта: стек, структуру директорий, основные модули backend и frontend, инварианты системы, что запрещено менять и какие части считаются критичными.

> Читай перед тем, как трогать любой файл проекта.

---

### 3. `ARCHITECTURE.md` — детальная архитектура
Углублённое описание модулей, зависимостей, потоков данных (invoice pipeline, дерево ТН ВЭД, NTM, расчёт платежей), взаимодействия frontend/backend, роутов API.

> Читай перед задачами, затрагивающими несколько модулей или потоков данных.

---

### 4. `KNOWN_PITFALLS.md` — известные ловушки
Журнал задокументированных особенностей и архитектурных ловушек проекта: pad-коды, L6/L8 синтез, legacy NTM, особенности SQLite/FTS5, исторические регрессии.

> Читай перед любым изменением в `tnved_catalog.py`, деревье ТН ВЭД или нетарифных мерах.

---

### 5. `CURRENT_STATE.md` — текущее состояние проекта
Что уже реализовано, активная задача, последние коммиты и архитектурные изменения, незавершённые задачи, текущие ограничения.

> Читай для понимания контекста: что было сделано до тебя и что сейчас в работе.

---

### 6. `ROADMAP.md` — техническая дорожная карта
Приоритизированный список технических задач с обоснованием по коду. Три уровня: высокий, средний, долгосрочный приоритет.

> Читай для выбора следующей задачи или понимания, что планируется в проекте.

---

## Правило обновления документов

| Событие | Что обновить |
|---|---|
| Завершена значимая задача | `CURRENT_STATE.md` |
| Найден новый сложный баг или ловушка | `KNOWN_PITFALLS.md` |
| Изменилась архитектура или добавлен модуль | `ARCHITECTURE.md`, `PROJECT_CONVENTIONS.md` |
| Закрыта или добавлена задача в роадмап | `ROADMAP.md` |
| Изменились правила работы | `ENGINEERING_PROTOCOL.md` |

## AI Team Infrastructure (этап 1)

Параллельный контур ролей, задач и протоколов:

| Путь | Назначение |
|------|------------|
| `.ai/VISION.md` | Продуктовая цель Tariff |
| `.ai/TASK_PROTOCOL.md` | Жизненный цикл задач, **запрет push без Ivan** |
| `.ai/QA_PROTOCOL.md` | QA с **фактическим выводом команд** |
| `.ai/PROMPTS.md` | Короткие команды для телефона |
| `.ai/agents/` | Роли: architect, backend, frontend, qa, reviewer, memory, PO |
| `.ai/tasks/` | Задачи (шаблон + TASK-SEMANTIC-003) |

Начни с `VISION.md` + `PROMPTS.md`, если нужна быстрая постановка задачи.

## Быстрый старт для нового агента

```
1. Прочитай .ai/ENGINEERING_PROTOCOL.md
2. Прочитай .ai/PROJECT_CONVENTIONS.md
3. Прочитай .ai/CURRENT_STATE.md
4. В зависимости от задачи — ARCHITECTURE.md и/или KNOWN_PITFALLS.md
5. Начинай работу по регламенту из п.1
```
