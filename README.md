# TN VED Pro & Eco-Fee — Шаги внедрения

## 0) Цели
- Классификация по описанию/фото + пояснения/примечания к разделам/группам.
- Дерево ТН ВЭД (как в ВЭД-Инфо) + ставки тарифа.
- Проверка нетарифных мер (ТР ТС/ЕАЭС/РФ) и расчёт эко-сбора.

## 1) Данные
- **Конфигурация**: `data/sources.yml` - официальные источники данных
- **Загрузка**: `python data/download_sources.py` - автоматическая загрузка
- **Специализированные скрипты**: `data/ingest/fetch_tariff_eec.py` - загрузка с сайта ЕЭК
- **Источники**: ЕЭК, ФТС, Минприроды (HTML/CSV/XLSX/XML/PDF)
- **Безопасность**: Только разрешенные домены, SHA256 проверки, retry логика

## 2) Backend
- FastAPI с эндпоинтами `/classify`, `/codes/*`, `/notes/*`, `/tariff/*`, `/non_tariff`, `/eco_fee`.
- БД: для старта SQLite, затем PostgreSQL.

## 3) AI-слой
- `backend/app/ai.py` — абстракция провайдера. Подключите OpenAI/Claude/Qwen/DeepSeek.
- Промпты в `ai/prompts/*`.

## 4) Frontend
- Flutter (stub в `frontend/flutter_stub/`). Позже — полноценный UI + загрузка Excel/фото.

## 5) Интеграции
- Загрузка Excel: парсер в backend (pandas) → классификация по строкам.
- Фото: OCR (pytesseract) + визуальный анализ модели.

## 6) Качество
- Юнит-тесты для парсера и правил.
- Логи ошибок, аудит решений ИИ.

## 7) Деплой
- Docker-compose для API + Postgres + Nginx.
