# Авторизация XML-API Альта-Софт

Оба описанных сервиса используют **MD5** в нижнем регистре (hex) и **UTF-8** для строк.

## Общее

1. `password_md5 = MD5(пароль)` — один раз от пароля в открытом виде.
2. В GET-запрос передаётся **`login`** открытым текстом и вычисленный **`secret`**.

Формулы **различаются** по первому компоненту строки перед подписью.

---

## «Товары и коды» (`/tik/xml/`)

```
secret = MD5( srchstr + ":" + login + ":" + password_md5 )
```

- `srchstr` — **то же значение**, что параметр `srchstr` в URL (строка поиска).

Источник: [API «Товары и коды»](http://wiki.alta.ru/index.php/API_%D0%BE%D0%BD%D0%BB%D0%B0%D0%B9%D0%BD-%D1%81%D0%B5%D1%80%D0%B2%D0%B8%D1%81%D0%B0_%D0%A2%D0%BE%D0%B2%D0%B0%D1%80%D1%8B_%D0%B8_%D0%9A%D0%BE%D0%B4%D1%8B).

## «Подбор кода» — шаг 2 (`/tnved/xml_apu/`)

Подсказки (шаг 1) с параметром `q` могут вызываться **без** логина.  
Для выдачи кодов по выбранной подсказке:

```
secret = MD5( payload_id + ":" + login + ":" + password_md5 )
```

- `payload_id` — значение элемента `payload` из ответа шага 1 (строка цифр).

Источник: [API «Подбор кода»](http://wiki.alta.ru/index.php/API_%D0%BE%D0%BD%D0%BB%D0%B0%D0%B9%D0%BD-%D1%81%D0%B5%D1%80%D0%B2%D0%B8%D1%81%D0%B0_%D0%9F%D0%BE%D0%B4%D0%B1%D0%BE%D1%80_%D0%BA%D0%BE%D0%B4%D0%B0).
