# XML-API «Подбор кода» / АПУ (Альта-Софт)

Официальная документация:  
http://wiki.alta.ru/index.php/API_%D0%BE%D0%BD%D0%BB%D0%B0%D0%B9%D0%BD-%D1%81%D0%B5%D1%80%D0%B2%D0%B8%D1%81%D0%B0_%D0%9F%D0%BE%D0%B4%D0%B1%D0%BE%D1%80_%D0%BA%D0%BE%D0%B4%D0%B0

## Назначение

Двухшаговый алфавитно-предметный указатель:

1. Подсказки-термины + идентификатор **`payload`** + группы ТН ВЭД.
2. Список кодов ТН ВЭД с **весами** и наименованиями по **`payload`** (нужны логин/пароль).

## Endpoint

- **URL:** `https://www.alta.ru/tnved/xml_apu/`
- **Метод:** `GET`

## Шаг 1 — подсказки

| Параметр | Обяз. | Описание |
|----------|--------|-----------|
| `q` | да | Поисковый текст |
| `limit` | нет | `-1` = без ограничения, иначе положительное число |

Пример (часто работает без авторизации):  
`https://www.alta.ru/tnved/xml_apu/?q=пилы`

### XML

Корень **`result`**, элементы **`line`**:

- `term` — текст подсказки  
- `tngroup` — перечень групп через запятую  
- `payload` — id для шага 2  
- `weight` — приоритет подсказки  

## Шаг 2 — коды по `payload`

| Параметр | Обяз. | Описание |
|----------|--------|-----------|
| `code` | да | Значение `payload` из шага 1 |
| `login` | да | Сервисный логин |
| `secret` | да | См. [alta_auth.md](./alta_auth.md) |
| `limit` | нет | Как на шаге 1; **`0` запрещён** (ошибка 201) |

### XML

Корень **`result`**, элементы **`line`**:

- `tnved` — код  
- `weight` — вес (релевантность)  
- `descr` — наименование  
- `descr_sh` — краткое описание  
- `tncode` — формат с пробелами `NNNN NN NNN N`  

## Тестовые данные (из wiki)

- Логин: `testlogin`, пароль: `testpassword`  
- Примеры `payload`: `112675973`, `114886448`  

## Переменные окружения (CustomsClear)

| Переменная | Назначение |
|------------|------------|
| `ALTA_APU_ENABLED` | `true` — прокси для шага 2 и объединённого сценария |
| `ALTA_APU_LOGIN` | Логин (может совпадать с ТиК) |
| `ALTA_APU_PASSWORD` | Пароль |
| `ALTA_APU_BASE_URL` | По умолчанию `https://www.alta.ru/tnved/xml_apu/` |

Шаг 1 (`q`) выполняется к тому же base URL и **не требует** секрета в типичном сценарии.
