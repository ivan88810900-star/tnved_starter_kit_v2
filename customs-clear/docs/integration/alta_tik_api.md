# XML-API «Товары и коды» (Альта-Софт)

Официальная документация:  
http://wiki.alta.ru/index.php/API_%D0%BE%D0%BD%D0%BB%D0%B0%D0%B9%D0%BD-%D1%81%D0%B5%D1%80%D0%B2%D0%B8%D1%81%D0%B0_%D0%A2%D0%BE%D0%B2%D0%B0%D1%80%D1%8B_%D0%B8_%D0%9A%D0%BE%D0%B4%D1%8B

## Назначение

По слову или словосочетанию — список кодов ТН ВЭД и до **100 записей на код** из практики декларирования. Следующие 100 по тому же коду — **отдельным запросом** (тарификация у Альты).

## Endpoint

- **URL:** `https://www.alta.ru/tik/xml/`
- **Метод:** `GET`

## Параметры

| Параметр | Обяз. | Описание |
|----------|--------|-----------|
| `srchstr` | да | Поисковая строка, **не короче 3 символов** |
| `login` | для данных | Сервисный логин |
| `secret` | для данных | См. [alta_auth.md](./alta_auth.md) |
| `tncode` | нет | Ограничение по подразделу ТН ВЭД (2–10 цифр), по wiki |
| `page` | нет | Страница дозапроса, **с 2** |

В примере на wiki для пагинации фигурирует параметр **`tnfiltr`** вместо `tncode` — при сбоях уточняйте у поддержки Альты.

## Успешный ответ (XML)

Фактическая структура ответа может быть вложенной:

- Корень: **`TikList`**
- Повторяющиеся блоки: **`TikInfo`**
  - **`Code`** — код ТН ВЭД
  - **`Count`** — число записей по коду
  - **`Notes`** → **`Note`** → **`Name`** — текст записи

## Ошибка (XML)

```xml
<Error>
  <ErrorCode>100</ErrorCode>
  <ErrorDescr>Пользователь не авторизирован</ErrorDescr>
</Error>
```

Коды 200, 210 и др. — в wiki.

## Переменные окружения (CustomsClear)

| Переменная | Назначение |
|------------|------------|
| `ALTA_TIK_ENABLED` | `true` — прокси-эндпоинты включены |
| `ALTA_TIK_LOGIN` | Логин |
| `ALTA_TIK_PASSWORD` | Пароль |
| `ALTA_TIK_BASE_URL` | По умолчанию `https://www.alta.ru/tik/xml/` |
