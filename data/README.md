# Источники данных TN VED Pro

## 📋 Обзор

Эта директория содержит конфигурацию и скрипты для загрузки официальных источников данных ТН ВЭД.

## 🗂️ Структура

```
data/
├── sources.yml              # Конфигурация источников данных
├── download_sources.py      # Скрипт загрузки данных
├── raw/                     # Загруженные файлы (создается автоматически)
└── README.md               # Эта документация
```

## ⚙️ Конфигурация (sources.yml)

### Разрешенные домены
```yaml
allowed_domains:
  - eec.eaeunion.org          # Евразийская экономическая комиссия
  - www.eurasiancommission.org # Евразийская комиссия
  - customs.gov.ru            # ФТС России
  - data.gov.ru              # Открытые данные РФ
  - minprirody.gov.ru        # Минприроды РФ
```

### Наборы данных

#### 1. tnved_tree - Дерево ТН ВЭД
- **Орган**: ЕЭК
- **Формат**: HTML
- **Содержит**: Коды, названия, иерархия

#### 2. section_notes - Примечания к разделам
- **Орган**: ЕЭК
- **Формат**: HTML
- **Содержит**: Примечания к разделам I-XXI

#### 3. chapter_notes - Примечания к группам
- **Орган**: ЕЭК
- **Формат**: HTML
- **Содержит**: Примечания к главам 01-97

#### 4. tariff_cet - Таможенный тариф
- **Орган**: ЕЭК/ФТС
- **Формат**: XLSX
- **Содержит**: Ставки пошлин, НДС, дополнительные сборы

#### 5. ntm_measures - Нетарифные меры
- **Орган**: ЕЭК/РФ
- **Формат**: XLSX
- **Содержит**: Ограничения, лицензии, сертификация

#### 6. eco_fee_rates - Эко-сбор
- **Орган**: РФ
- **Формат**: XLSX
- **Содержит**: Ставки экологического сбора

## 🚀 Использование

### Загрузка всех данных
```bash
cd data/
python download_sources.py
```

### Загрузка конкретного набора
```bash
python download_sources.py --dataset tnved_tree
```

### Просмотр списка наборов
```bash
python download_sources.py --list
```

### Проверка целостности
```bash
python download_sources.py --verify
```

## 🔧 Настройка источников

### 1. Обновление URL
Отредактируйте `sources.yml`:
```yaml
datasets:
  tnved_tree:
    urls:
      - "https://eec.eaeunion.org/actual-link-to-data"
```

### 2. Добавление нового источника
```yaml
datasets:
  new_dataset:
    name: "Новый источник"
    authority: "Орган"
    version: "2025-01-01"
    urls:
      - "https://example.com/data.xlsx"
    file_type: "XLSX"
    save_as: "new_data_2025-01-01.xlsx"
    checksum: ""
```

### 3. Обновление версии
```yaml
datasets:
  tnved_tree:
    version: "2025-02-01"  # Новая версия
    save_as: "tnved_tree_2025-02-01.html"  # Новое имя файла
```

## 🔐 Безопасность

### Проверка доменов
Скрипт загружает данные только с разрешенных доменов:
- ✅ `eec.eaeunion.org` - официальный сайт ЕЭК
- ✅ `customs.gov.ru` - официальный сайт ФТС
- ❌ Любые другие домены блокируются

### Проверка целостности
- **SHA256** хеши для всех файлов
- **Автоматический расчет** при загрузке
- **Проверка целостности** перед использованием

### Логирование
```bash
# Просмотр логов загрузки
python download_sources.py 2>&1 | tee download.log
```

## 🐛 Устранение проблем

### Ошибка домена
```
❌ Домен example.com не разрешен
```
**Решение**: Добавьте домен в `allowed_domains` или используйте разрешенный домен.

### Ошибка загрузки
```
❌ Ошибка загрузки https://example.com: 404
```
**Решение**: Проверьте URL в конфигурации.

### Ошибка checksum
```
❌ tnved_tree: checksum не совпадает
```
**Решение**: Перезагрузите файл или обновите checksum в конфигурации.

### Таймаут загрузки
```
❌ Ошибка загрузки: timeout
```
**Решение**: Увеличьте timeout в скрипте или проверьте интернет-соединение.

## 📈 Метрики

- **eco_fee_rates.xlsx**: ~1-2 MB

### Время загрузки
- **Быстрое соединение**: 1-2 минуты
- **Медленное соединение**: 5-10 минут
- **Повторная загрузка**: несколько секунд (кеш)

## 🔄 Обновление данных

### Ежемесячное обновление
1. Проверьте новые версии на официальных сайтах
2. Обновите URL в `sources.yml`
3. Запустите загрузку: `python download_sources.py`
4. Проверьте целостность: `python download_sources.py --verify`

### Экстренное обновление
```bash
# Принудительная перезагрузка
rm -rf raw/*
python download_sources.py
```

## 📞 Поддержка

При проблемах с загрузкой данных:
1. Проверьте интернет-соединение
2. Убедитесь в доступности официальных сайтов
3. Проверьте актуальность URL в конфигурации
4. Обратитесь к логам загрузки для диагностики
